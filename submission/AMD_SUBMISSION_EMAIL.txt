APPROACH AT A GLANCE
----------------------------------------------------------------

  Lever:   Per-Phase Relaxed-Acceptance MTP (TRT-LLM "thinking" port)
  Source:  3 ATOM files modified (+202 LOC, 0 removed):
             - atom/utils/envs.py                   (+8)   env flag
             - atom/model_engine/model_runner.py    (+30)  GPU phase tensor + plumbing
             - atom/model_ops/rejection_sampler.py  (+164) Triton kernel + dispatcher

  Mechanism:
    A per-sequence int8 phase tensor (0=NOT_THINKING / 1=THINKING / 2=DONE)
    tracks whether each request is inside a <think>...</think> reasoning block.
    A new Triton kernel `rejection_phased_sample_kernel` branches per-request:
      inside  <think>...</think>:  accept if draft in top-N=10  (relaxed_delta=0.6)
      outside <think>...</think>:  accept if draft in top-N=8   (= baseline)
    The kernel scans committed tokens for the DeepSeek-R1 <think>/</think> token
    IDs (128798 / 128799) and updates phase via tl.store -- no Python setattr in
    the forward path, so the kernel stays cudagraph-safe.

  Why this works:
    Reasoning-phase tokens have a flatter target distribution -> top-N=10
    accepts more drafts without harming GSM8K accuracy. Answer-phase tokens are
    peakier -> stay at top-N=8 to preserve correctness. Result: same draft
    network, higher acceptance only where it's safe -> lower TPOT -> +1 gate.

  Activation:
    export ATOM_ENABLE_PER_PHASE_RELAXED_MTP=1
    (default 0 = bit-identical to upstream)
